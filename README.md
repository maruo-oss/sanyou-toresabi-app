# かんばん照合アプリ（GAS Webアプリ版）

## 概要
eかんばんの付け間違い・出荷間違いを防止するためのQRコード照合アプリです。  
Google Apps Script (GAS) の Web アプリとして動作し、iPhoneのブラウザ（Safari）から利用できます。

## 目的
- eかんばんと切断仕上票のQRコードを照合し、異材・異品種混入を防止する
- eかんばん枝番と切断仕上No.のデータ紐づけを記録する
- トレーサビリティの強化（BigQuery + Looker Studioでの確認）

#### 読み取りフロー（ワンボタン操作）
1. 「読み取り開始」ボタンを押す
2. カメラ起動 → **eかんばん**のQRコードをスキャン
3. 製品型番・規定本数（個数）を自動抽出
4. カメラ自動再起動 → **切断仕上票**のQRコードをスキャン
5. 規定本数に満たなければ再度カメラ起動（**最大3回まで**）
6. 製品型番を自動照合
   - **一致** → 成功メッセージ表示 + データベースに保存
   - **不一致** → エラーアラート表示（保存しない）

## QRコード照合の仕組み

### 照合キー: 製品型番
両方のQRコードから**製品型番**（例: `JGF02-001000-31`）を抽出し、一致を確認します。

```
eかんばんQR:     B02 [JGF02-001000-31] ...
                  ↑個数  ↑製品型番
                         ║
                    照合（一致確認）
                         ║
切断仕上票QR:    [JGF02-001000-31] 20260303 00278389 ...
                  ↑製品型番          ↑日付     ↑仕上NO
```

### 規定本数の判定
eかんばんQRの先頭部分に含まれる**個数**（例: `B02` → 2個）により、  
切断仕上票を何枚読み取る必要があるかを自動判定します。

## システム構成

```
┌─────────────┐     ┌──────────────────────────┐
│  iPhone      │     │  Google Apps Script       │
│  (Safari)    │────→│  ├── Code.gs (メイン)     │
│              │     │  ├── BigQuerySync.gs      │
│  QRスキャン   │     │  ├── index.html           │
│  (jsQR)     │     │  ├── scanner.html          │
│              │     │  ├── list.html             │
└─────────────┘     │  ├── css.html              │
                     │  └── js.html               │
                     └──────┬───────────┬─────────┘
                            │           │
                     ┌──────▼──┐  ┌─────▼──────┐
                     │SpreadSheet│  │  BigQuery   │
                     │(即時保存) │  │(5分バッチ)  │
                     │          │  │             │
                     │・照合データ│  │・照合データ  │
                     │・社員マスタ│  │             │
                     └──────────┘  └──────┬──────┘
                                          │
                                   ┌──────▼──────┐
                                   │ Looker Studio│
                                   │ (詳細確認)   │
                                   └─────────────┘
```

### データの流れ
1. **即時保存**: iPhone → GAS → スプレッドシート（1秒以内）
2. **バッチ転送**: スプレッドシート → BigQuery（5分間隔の自動トリガー）
3. **分析**: BigQuery → Looker Studio（PCで詳細データ確認）

## 保存データ項目

| 項目 | 説明 | 取得元 |
|------|------|--------|
| タイムスタンプ | 照合日時 | 自動記録 |
| 社員番号 | 作業者のID | 手入力 |
| 出荷日 | 出荷予定日 | カレンダー選択 |
| 出荷便 | T1 / T2 / T3 / T4 | プルダウン選択 |
| 背番号 | 製品の背番号 | 切断仕上票QRから抽出 |
| 製品型番 | 照合キー | QRから抽出 |
| eかんばん枝番 | かんばん通番 | eかんばんQRから抽出 |
| 切断仕上No.1 | 1枚目 | 切断仕上票QRから抽出 |
| 切断仕上No.2 | 2枚目（あれば） | 切断仕上票QRから抽出 |
| 切断仕上No.3 | 3枚目（あれば） | 切断仕上票QRから抽出 |

## 使用デバイス・環境
- **作業端末**: iPhone（Safari）
- **確認端末**: PC（Looker Studio）
- **動作環境**: 工場内（暗所対応: LED点灯機能あり）
- **操作性**: 手袋着用・片手操作を想定したワンボタン設計
