要件定義書：かんばん照合アプリ（現行仕様）

最終更新日: 2026-04-22


1. プロジェクト概要

システム名称： かんばん照合アプリ
目的：
  ・eかんばんの付け間違いおよび出荷間違いの防止
  ・eかんばん内の枝番と切断仕上No.のデータ紐づけ
  ・異材・異品種混入の防止
  ・トレーサビリティの強化（客先からの要望）
背景： 手作業によるeかんばん添付作業において、異材・異品種混入のリスクが
      発生しており、デジタル照合による確実な検品体制を構築する。

プラットフォーム：
  ・Google Apps Script（GAS）によるWebアプリ
  ・フロントエンド：HTML5 / CSS / JavaScript
  ・データ保存：Google スプレッドシート（即時保存）＋BigQuery（5分バッチ転送）


2. 業務フロー

準備：
  作業者がアプリを起動し、社員番号・出荷日・出荷便を設定。
  社員名は社員マスタから自動検索。退社済み社員（N列=TRUE）はスキャン画面へ進めない。

現品作業：
  ケースに付いている「切断仕上票」を確認し、対応する「eかんばん」を設置。

デジタル照合（本アプリ）：
  ① 「eかんばん」のQRコードをスキャン
  ② 「切断仕上票」のQRコードをスキャン
     ・必要枚数に達するまで連続してスキャン（最大3枚）
     ・同じ切断仕上No.の重複読み取りはエラーにし、再スキャン可

判定： アプリが以下を確認し、全てOKなら実績として保存
  ・eかんばんと切断仕上票の「製品型番」が一致
  ・複数枚の切断仕上No.に重複がない
  ・eかんばんの「収容数」と切断仕上票の「本数」が一致

データ管理：
  リスト検索画面から過去の登録データを検索・修正・削除可能。
  日時の新しい順で表示。

最終確認：
  アプリ上の集計結果と、客先納品書の数字を照らし合わせ差異がないか目視確認する。


3. 画面構成と機能要件

3-1. トップ画面（index.html） 確認者登録

  ・社員コード入力（テンキー入力）
  ・社員名・工場の自動表示（社員マスタ参照）
    - 退社済み社員（N列=TRUE）は「退社済み」表示、読み取り画面ボタンは無効
    - 該当なしの場合もボタン無効
  ・出荷日選択（カレンダーUI、デフォルト当日）
  ・出荷便選択（T1〜T4 プルダウン）
  ・LED設定（ON/OFF）
  ・読み取り画面ボタン（社員認証OK後に有効化）
  ・リスト検索ボタン


3-2. 読み取り画面（scanner.html） QRスキャン・照合

  ヘッダー：
    ・戻る / リスト検索 / ホーム の各ボタン

  カメラ制御：
    ・iPhone 背面カメラを起動し、写真撮影でQRを取り込む方式
    ・撮影画像から qr-scanner ＋ jsQR ライブラリでQR検出（フォールバック対応）

  eかんばん読み取り：
    ・ボタン押下でカメラ起動 → 撮影 → QR解析
    ・抽出項目: タイプ / 個数（規定枚数）/ 製品型番 / 背番号 / 枝番 / 収容数
    ・切断仕上票のQRを誤って読み取った場合はエラー表示

  切断仕上票読み取り：
    ・eかんばんの「個数」に達するまで連続スキャン（最大3枚）
    ・抽出項目: 製品型番 / 日付 / 仕上NO / 切断NO / 背番号 / 本数
    ・eかんばんを誤って読み取った場合はエラー表示

  切断仕上No.重複チェック：
    ・スキャン時に既読のNoと同じなら即座にエラー表示し、再スキャン可能
    ・状態は維持されるためeかんばんからやり直す必要なし

  照合ロジック：
    ・全切断仕上票の製品型番がeかんばんと一致 ＋
    ・切断仕上No.重複なし ＋
    ・eかんばん収容数＝切断仕上票本数
    → 全てOKなら「照合OK」→ スプレッドシートに保存
    → いずれかNGの場合はアラート表示、保存せず

  保存完了後：
    ・アラート「保存しました」→ OKで「次の読み取り」ボタンが出現
    ・「次の読み取り」で新しいeかんばんからサイクル開始

  デバッグ情報：
    ・スキャン成功時にQR生データを画面下部に表示（解析・検証用）


3-3. リスト検索画面（list.html） データ閲覧・修正・削除

  ヘッダー：
    ・戻る / ホーム の各ボタン

  検索フィルタ：
    ・出荷日（デフォルト当日、「全期間」ボタンで解除可）
    ・出荷便（T1〜T4、「すべて」選択可）
    ・フリーテキスト検索（背番号 / 製品型番 / 枝番 / 切断No / 社員番号）
      入力中にリアルタイム絞り込み

  集計表示：
    ・出荷日×出荷便でフィルタし、背番号ごとのケース数合計を自動集計
    ・客先納品書の数量との突き合わせに使用

  詳細リスト表示：
    ・日時の新しい順（降順）で表示
    ・各レコードに「修正」「削除」ボタン

  修正機能：
    ・モーダルで全9項目を編集（社員番号 / 出荷日 / 出荷便 / 背番号 /
      製品型番 / eかんばん枝番 / 切断仕上No.①②③）
    ・保存でスプレッドシート更新 → リスト再描画

  削除機能：
    ・確認モーダルを経由してレコード削除
    ・削除後リスト再描画


4. 非機能要件

  使用デバイス： iPhone（Safari）/ PC（Looker Studio閲覧用）
  配信方式：    GAS Web App（HTTPS）
  環境条件：    工場内の暗所でもLED点灯により安定してスキャンできること
  操作性：      手袋着用・片手操作前提
  動作スピード： 保存処理は 2 秒以内を目標
  データ保持：  Google スプレッドシート（即時）＋ BigQuery（5分バッチ）


5. データ構造（保存項目）

  A列: タイムスタンプ         自動記録
  B列: 社員番号               確認者
  C列: 出荷日                 カレンダーUIで選択
  D列: 出荷便                 T1〜T4
  E列: 背番号                 切断仕上票QRから抽出
  F列: 製品型番               照合キー
  G列: eかんばん枝番          eかんばんQRから抽出
  H列: 切断仕上No.①          1枚目
  I列: 切断仕上No.②          2枚目（必要時）
  J列: 切断仕上No.③          3枚目（必要時）

  ※ 重複登録防止：
    同一「eかんばん枝番 + 出荷日 + 出荷便」の組み合わせは再登録不可


6. マスタ構造

6-1. 社員マスタ（別スプレッドシート、スクリプトプロパティで参照）
  A列: 社員番号
  B列: 氏名
  C列: 工場
  N列: 退社フラグ（TRUE=退社済・非表示、FALSE=在籍・表示）


7. QRコード仕様（参考）

7-1. eかんばんQR（例）
  B02JGF02-001000-31     00000006006                       07845 0000006399105700       282

  抽出ルール：
    位置 0:    タイプ（1文字）
    位置 1-2:  個数（2桁。規定枚数、複数回スキャンに使用）
    位置 3-18: 製品型番（16文字、トリム）
    空白区切り2番目のフィールド末尾4文字のうち先頭2文字：収容数
    空白区切りで最初に現れる5桁数字: eかんばん枝番
    空白区切り最終フィールド: 背番号

7-2. 切断仕上票QR（例）
  JGF02-001000-31202603030027838900281478         E586442       282 0007020000602001000251000001

  抽出ルール：
    先頭〜日付パターン(20YYMMDD)の直前: 製品型番
    日付後 8文字: 仕上NO
    さらに 8文字: 切断NO
    残りから 1〜4桁の数字: 背番号
    残りの中の最後の12桁以上の数字ブロックの 7〜12 文字目: 本数
